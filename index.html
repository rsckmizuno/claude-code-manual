Section 01

Claude Codeとは？
ChatGPTとの決定的な違い

まず「ChatGPTと何が違うの？」という疑問を解消します。この違いを腑に落とすだけで、Claude Codeの使い方が自然と分かるようになります。

基本概念 「優秀な相談相手」と「優秀な部下」の違い
従来型AI / 相談相手モデル

ChatGPT

  • ヒント・文章案を提案してくれる
  • 答えを出したら終わり
  • コピペして自分で実行する必要がある
  • ツールの切り替えは人間がやる
  • 「調べてくれる」が「動いてくれない」
次世代AI / 実行エージェントモデル

Claude Code

  • 自らファイルを作成・保存・送信する
  • 指示したタスクを最後まで完遂する
  • 他のツールと自動で連携する
  • 複数工程を自律的にこなす
  • 「動いてくれる」優秀な部下
💡 たとえ話で理解：
ChatGPTは「優秀なコンサルタント」。相談すると良いアドバイスをくれますが、実行は自分がやります。
Claude Codeは「優秀な社員」。指示を出すと自分でパソコンを操作し、調べて、ファイルを作り、送信まで完了させます。
はじめ方 難しそうに見えるけど、実は誰でも使える
!

「黒い画面が必要」は昔の話

今は普通のチャット画面から使えます

以前はエンジニアが「ターミナル（黒い画面にコマンドを打ち込む画面）」を使う必要がありましたが、現在はLINEやSlackのようなチャット画面から日本語で指示するだけで使えます。

OK 「プログラミングを知らなくても使える」が今のClaude Codeの前提です。必要なのは日本語で指示を出す力だけです。
2日 → 3時間 40ページ規模の企画書・制作資料の作成時間が劇的に短縮された実例
コード不要 プログラミング知識ゼロでOK。日本語チャットだけで操作できる
Section 02

5体のAIエージェントによる
分業体制

Claude Codeの最大の特徴は、複数のAIに役割を分担させられること。会議の議事録を渡すだけで、5体のAIが連携して企画書を自動で作り上げます。

仕組みを理解 なぜ1体ではなく5体に分けるのか？
🧠 AIの「記憶容量」には上限がある：
AIには一度に覚えられる情報量（コンテキスト）に限界があります。1体に全部やらせると、途中で内容を「忘れて」精度が下がります。

→ だから役割を細分化して、それぞれの専門AIに担当させることで、精度を高く保てます。
これは人間のチームと同じです。1人に全部やらせず、得意な人に分担させるほうが良い仕事になります。

02

実践手順 議事録 → 企画書の作り方（ステップバイステップ）
1

会議の議事録を用意する

テキストデータになっていれば何でもOK

会議のメモ、録音の文字起こし、Notionのメモなど、テキスト形式になっているものを用意します。

  • Wordファイル・テキストファイル・メモ帳のテキストなど、文字が書いてあるものならOKです
  • 手書きメモの場合は、スマホで写真を撮ってGoogleドキュメントに貼り付けるとテキスト化できます（Googleドキュメント → 挿入 → 画像からテキスト）
  • 録音がある場合はClaude Codeに「この音声を文字起こしして」とお願いすることもできます
2

Claude Codeを開いてチャット画面に貼り付ける

LINEに文章を送る感覚でOK

Claude Codeのチャット画面を開き、議事録のテキストをコピー＆ペーストします。

  • claude.ai にアクセスしてアカウントを作成・ログインします（Googleアカウントでサインインできます）
  • 「Claude Code」のプランを選択します（有料プランが必要です）
  • チャット欄に議事録のテキストをそのまま貼り付けます
POINT ファイルをそのままドラッグ＆ドロップで添付することも可能です。
3

指示文（プロンプト）を入力して送信する

この「言い方」が成果を左右します

議事録を貼り付けた後、以下のような指示文を追加して送信します。

上記は先週の営業会議の議事録です。
以下の役割を持つ5つのAIエージェントを順番に動かして、来月の経営会議で使える企画書（A4・10ページ程度）を作成してください。

① 論点分析：課題と論点を整理
② 市場リサーチ：競合・業界トレンドを調査
③ 戦略考案：3つの戦略オプションを提案
④ 批判的レビュー：各戦略のリスクを洗い出し
⑤ 最終レポート：上記をまとめて企画書を作成
注意 役割を番号で明示することがポイントです。曖昧な指示だとAIが役割を省略することがあります。
4

AIが自動で処理するのを待つ

途中で操作する必要はありません

送信後、Claude Codeが5つの役割を順番に実行し始めます。各AIが処理している様子が画面に表示されます。

  • 処理中は画面に「考え中...」などの表示が出ます。そのまま待ちましょう
  • 他のウィンドウで別の仕事をしながら待っても大丈夫です
  • 40ページ規模の資料なら、おおよそ1〜3時間で完成します
OK 途中の処理が見えるので、方向性が違う場合は「ちょっと待って、③の戦略は〇〇の方向で考えて」と途中で修正指示を出すことも可能です。
5

完成した企画書を確認・修正する

人間の判断・確認が最後の仕事

企画書が完成したら、内容を確認します。AIはあくまでも「たたき台」を作るので、最後は人間が確認・修正します。

  • 数字・固有名詞・事実確認は必ず人間が目視でチェックします
  • 修正したい箇所は「3ページ目の市場規模の数値を2024年の最新データに差し替えて」など具体的に指示します
  • Word・PDFへの変換も「Wordファイルとして書き出して」と指示するだけで完了します
重要 AIが出したデータや数値は必ず一次情報で確認してください。AIは古いデータや誤った情報を自信満々で出すことがあります。
Section 03

Skills（スキルズ）機能で
業務を自動化する

うまくいった作業の手順をAIに「型」として記憶させる機能です。次回からは短い言葉だけで同じ品質の作業を再現できます。プログラミングは一切不要です。

仕組みを理解 「スキル」って何？マクロ記録に近い概念
📌 Excelのマクロをイメージしてください：
Excelでは「毎回同じ操作をマクロとして記録」しておくと、次回はボタン1つで同じことができます。
Skillsはそれと同じです。AIが実行した手順を「スキル」として保存しておくと、次回は短い言葉だけで再現できます。

しかもコードを書く必要が一切ありません。「これをスキル化して」と言うだけでAIが自動で設定してくれます。
実践手順 スキルを作るまでの具体的な流れ
1

まず普通に作業を指示してみる

最初は「試し」のつもりで気軽に

スキル化を意識せず、まずは普通に仕事を頼んでみます。

この資料にグラフと図解を追加して、経営者向けに見やすく整えてください。フォントはゴシック体で統一、カラーはブランドカラーの#1a4d8fを使ってください。
  • 指示は具体的なほど良い結果が得られます（「見やすく」より「フォント・色・グラフ形式を指定」）
  • 最初の1〜2回は結果を確認しながら「もう少し大きいフォントに」など調整しましょう
  • 「これだ！」という仕上がりになるまで何度か修正をかけます
2

納得できたらスキル化を指示する

この一言だけでOK

完成した作業の品質に満足したら、すぐにスキル化を指示します。

この作業の手順をスキルとして保存しておいてください。次回から「資料を整える」と言ったら、同じフォーマットで整えてください。
  • Claude Codeが自動で手順を分析してスキルとして保存します（コード不要）
  • 「スキル名」を指定しておくと後で呼び出しやすいです（例：「経営者向け資料フォーマット」）
  • 保存が完了すると「スキルとして保存しました」と表示されます
POINT スキルには「例外処理」も教えられます。「図が入ってない場合はシンプルにテキストだけで整えて」などの条件も追加しておくとより万能になります。
3

次回から短い言葉だけで再現する

AIが空気を読んで動いてくれる

スキルを保存した後は、次回から簡単な言葉で同じ品質の作業が再現されます。

この資料、さっきのスキルで整えておいて。
  • AIが保存した手順を思い出し、同じフォーマット・ルールで作業を実行します
  • 1回目に20分かけて細かく指示した内容が、次回から3秒の指示で完了します
  • スキルは蓄積するほど便利になるため、うまくいった作業はすぐスキル化する習慣をつけましょう
4

スキルを改善・更新する

育てるほど賢くなる

スキルは一度作って終わりではなく、使いながら改善できます。

  • 使ってみて「少し違う」と感じたら「さっきのスキルに〇〇のルールも追加して」と指示するだけで更新できます
  • 季節や状況に合わせて「Q4は数字中心のレイアウトに変えて」など条件分岐も追加できます
  • チームメンバーにスキルを共有する場合は「このスキルの設定をテキストで書き出して」と指示すると共有できます
活用例 まずこれをスキル化しよう — すぐ使えるリスト

🗂 スキル化しておきたい業務トップ7

  • 週次レポート作成：KPIデータを貼り付けるだけで定型フォーマットに整形
  • 議事録の論点整理：会議メモをアクションアイテム形式に変換
  • 競合調査フォーマット：競合名を渡すだけで同じ比較表を自動生成
  • メール文面のトーン調整：「取引先向け丁寧バージョン」に統一変換
  • 朝のタスク優先順位付け：タスクリストを渡すと重要度×緊急度で並び替え
  • SNS投稿の文体統一：ブランドトーンに合わせた文体で自動リライト
  • 提案書フォーマット適用：会社の提案書テンプレートに内容を流し込み
Section 04

3つの運用ポイント

Claude Codeを最大限に活かすための、実務ですぐ使える3つの運用方法を、初めての方でも迷わないよう具体的な手順で解説します。

運用① 会社の情報をフォルダ構造で整理してアクセス権を渡す
📁 考え方：Claude Codeを「会社の内情を知っている優秀な社員」にするには、会社の情報へのアクセスを与える必要があります。フォルダを整理してアクセス権を付与するだけで、部門横断的な提案や予算立てができるようになります。
1

フォルダ構造を決める

会社の組織図をフォルダに置き換えるイメージ

Google DriveやOneDriveに以下のようなフォルダ構造を作ります。

  • 📂 会社情報 / 01_経営戦略（3ヶ年計画・MVV・KPIなど）
  • 📂 会社情報 / 02_マーケティング（顧客データ・競合分析・キャンペーン）
  • 📂 会社情報 / 03_セールス（商談履歴・提案書・価格表）
  • 📂 会社情報 / 04_財務（予算・実績・コスト構造）
  • 📂 会社情報 / 05_人事（組織図・評価基準）
注意 機密性の高い情報（個人情報・未公開の財務情報）はフォルダに入れないよう注意してください。
2

Claude Codeにフォルダへのアクセスを許可する

「読んでいいよ」と教える

Claude CodeにGoogle DriveやOneDriveとの連携を設定し、上記フォルダへのアクセスを許可します。

  • Claude Codeの設定画面から「ツール連携」または「Integrations」を開きます
  • Google Driveを選択し、Googleアカウントでの連携を承認します
  • アクセスを許可するフォルダを「会社情報」フォルダに指定します
「会社情報」フォルダの内容を読んで、我々の経営戦略・売上・コスト構造を理解した上で、来期の予算配分案を作成してください。
3

部門横断的な提案を依頼する

複数部門の情報を統合した判断ができる

フォルダを読んだClaude Codeは、各部門の情報を統合した上で回答・提案をします。

  • 「マーケ予算を10%削減した場合に売上への影響を試算して」などの複雑な質問に答えられるようになります
  • 新規事業の提案時に「今の人員構成・財務状況を踏まえた現実的な計画」を出せます
  • 定期的にフォルダの内容を更新することで、常に最新情報をもとに提案してくれます
運用② 他のAI・ツールと連携させて「一元管理」する
🔗 考え方：今まで「Googleカレンダーを開く→Slackを確認する→Gmailを返信する→資料を作る」と人間がツールを行ったり来たりしていました。Claude Codeを「ハブ（中継点）」にすることで、人間はClaude Codeだけ見ていれば全てのツールを操作できるようになります。
1

連携するツールを設定する

まず1〜2個から始めるのがおすすめ

Claude Codeの設定画面から使用するツールを連携します。最初はよく使う1〜2つから始めましょう。

  • Claude Code設定 →「ツール連携」→ 連携したいサービスを選択
  • 各サービスのアカウントでログインして連携を承認するだけです
  • おすすめの最初の連携：①Googleカレンダー ②Gmail（この2つで大幅に楽になります）
2

日本語で「横断的な」指示を出す

複数ツールをまたぐ作業を1つの指示で完結

連携後は、複数のツールをまたぐ作業を1つの指示で依頼できます。

今日のカレンダーを確認して、午後の〇〇社との打ち合わせ前に送るべき準備資料の送付漏れがないかチェックして。もし漏れていたら下書きメールを作って。
  • Claude Codeがカレンダーを確認 → 打ち合わせを特定 → Gmailで過去メールを確認 → 下書き作成まで自動実行します
  • 各ツールを切り替える必要がなくなり、認知負荷が大幅に下がります

🔧 連携できる主なツール一覧

Claude Codeが「ハブ」として連携できる代表的なツールです。

  • Googleカレンダー：スケジュール確認・登録・リマインド設定
  • Gmail：メール下書き・送信・分類・返信指示
  • Slack：メッセージ送信・通知・要約
  • Notion：ドキュメント作成・整理・データベース更新
  • Google Drive：ファイル作成・整理・共有
  • 他の生成AI：Gemini・GPT-4などと役割分担して使うことも可能
運用③ 複数タブで複数プロジェクトを同時進行する
⚡ 考え方：Claude Codeはタブを複数開くと、それぞれのタブで別々のタスクを同時進行できます。人間は各タブに指示を出したら、AIが処理している間に別の仕事ができます。実質1人で3〜5人分の仕事を回しているような状態になります。
1

タブを複数開いて別々のタスクを割り当てる

ブラウザのタブを増やすだけ

ブラウザで claude.ai の新しいタブを複数開き、それぞれ別の会話を始めます。

  • タブ①：「新商品の企画書を作って」と指示して処理を開始
  • タブ②：「競合A社の最新動向をリサーチして」と指示して処理を開始
  • タブ③：「来週のプレゼン用スライド構成を考えて」と指示して処理を開始
  • それぞれのAIが並列で動くため、3つのタスクが同時進行します
POINT タスクを投げたら、そのタブは最小化して別の仕事をしましょう。AIの処理が終わる頃に戻ってくれば効率的です。
2

処理が終わったタブから順に確認・修正する

マネージャーとして各タブの成果を確認する

各タブのAIが処理を完了したら、順番に確認して必要な修正指示を追加します。

  • 完了したタブに「いいね。ただし〇〇の部分を修正して」と追加指示を出す
  • 修正指示を出したら次のタブに移動して、そちらも確認する
  • このループを回すことで、常に複数のプロジェクトが動いている状態を維持できます
Section 05

人間に求められるのは
「言語化力」

AIエージェントをうまく使いこなせる人とそうでない人の差は、プログラミングの知識でも技術力でもありません。「自分の要求を正確に言葉にする力」だけです。

考え方 AIマネジメント ＝ 人間マネジメント
💬 AIへの指示は「部下への指示」と同じです：
「いい感じにやっといて」という指示では、優秀な部下でも困ります。
「〇〇という目的のために、△△という対象に向けて、□□という形式で作成して」という明確な指示があれば、誰でも動けます。

これは人間でもAIでも変わりません。つまり管理職・経営者ほど、この力が活きるツールです。
良い指示の公式
目的（何のために）　＋　対象（誰に向けて）
＋　制約（どんな条件で）　＋　形式（どんなアウトプットで）
実践 良い指示・改善が必要な指示の比較
✅ 良い指示の例

「競合A社・B社・C社と比較した自社の強み・弱みをまとめて、来月の営業会議で使える提案書（A4・4ページ）を作って。ターゲットは40代・製造業の経営者。専門用語は使わずシンプルに」

目的・対象・制約・形式がすべて明示されているため、AIが迷わず動ける

❌ 改善が必要な指示の例

「いい感じの提案書を作って」

目的・対象・制約・形式が全て不明。AIが推測で動くため、期待と大きく外れた結果になりやすい

✅ 良い修正指示の例

「3ページ目のグラフをもっとシンプルにして。データ項目を3つだけに絞って、棒グラフではなく円グラフに変えて」

何を・どう変えるかが具体的なので、意図通りに修正される

❌ 改善が必要な修正指示の例

「グラフをもっと分かりやすくして」

「分かりやすい」の基準が人によって異なる。AIの判断任せになり、さらに複雑になることもある

スキルアップ 言語化力を鍛える4つのコツ

① 「何のために」を最初に書く

「〇〇会議で使う」「△△という課題を解決するために」と目的を最初に書くと、AIが全体の方向性を理解しやすくなります。

② 「誰が読む・使う」を明示する

「60代・非IT系の経営者向け」「入社1年目の営業担当向け」など読み手を具体的に伝えると、トーン・用語レベルが自動で調整されます。

③ 分量・形式を数字で指定する

「見やすく」ではなく「A4・3ページ・箇条書き中心」など数字と形式で指定します。数字があると判断基準が明確になります。

④ 「やってほしくないこと」も伝える

「専門用語は使わないで」「グラフは入れないで」などの禁止事項を明示することで、意図しない方向への暴走を防げます。
